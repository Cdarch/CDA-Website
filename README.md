# א.ח.ד אדריכלים — מדריך עריכת האתר

ברוך הבא, חיים! זה ה-repository (תיקיית הקבצים) של האתר שלך.
**כל מה שכתוב כאן הוא בעברית, ולא צריך לדעת קוד כדי לערוך.**

לאחר כל שינוי שתשמור — האתר באוויר יתעדכן תוך כדקה אוטומטית.

---

## מבנה האתר

הקבצים החשובים שתערוך:

| קובץ | מה יש בו |
|---|---|
| `projects.js` | **הקובץ הראשי** — כל הפרויקטים, היזמים, ופרטיהם |
| `assets/projects/` | תמונות הפרויקטים |
| `assets/clients/` | לוגואים של היזמים (אזורים, אבן דרך וכו') |
| `assets/logos/ahd-logo.jpeg` | הלוגו של המשרד |

קבצים שאסור לגעת בהם (קוד טכני):
- `index.html`, `app.js`, `styles.css`, `.htaccess`, `robots.txt`

## איך עורכים פרויקט קיים

נניח שאתה רוצה לעדכן את "ויצמן 105" — לשנות את מספר היחידות מ-21 ל-24.

### צעדים בדפדפן

1. נכנס ל-GitHub ופותח את ה-repository של האתר.
2. לוחץ על הקובץ **`projects.js`**.
3. לוחץ על אייקון העיפרון בפינה הימנית-עליונה של הקובץ ("Edit this file").
4. מחפש את השורה של הפרויקט (Ctrl+F → "ויצמן" או "weizmann"):
   ```
   units: '21 יח"ד',
   ```
5. משנה ל:
   ```
   units: '24 יח"ד',
   ```
   **חשוב:** להשאיר את הגרשיים והפסיק כפי שהם.
6. גולל למטה, לוחץ על **"Commit changes..."** (כפתור ירוק).
7. בחלון שנפתח: כותב תיאור קצר ("עדכון מספר יחידות בויצמן 105") ולוחץ "Commit changes" שוב.
8. תוך כ-60 שניות האתר באוויר יתעדכן.

### כללי בטיחות לעריכת `projects.js`

- כל שורה חייבת להסתיים בפסיק `,` (חוץ מהשורה האחרונה לפני סוגריים `]`).
- ערכים בעברית — בתוך גרשיים בודדים: `'...'`.
- אם יש בערך גרש בודד (למשל "ז'בוטינסקי"), השתמש בלוכסן הפוך לפניו: `'ז\'בוטינסקי'`.
- אם יש בערך מירכאות כפולות (למשל יח"ד), אין בעיה — הן בתוך גרשיים בודדים.

## איך מוסיפים פרויקט חדש

### צעד 1: מעלים את תמונת הפרויקט

1. ב-GitHub, נכנס לתיקייה `assets/projects/`.
2. לוחץ על "Add file" → "Upload files".
3. גורר את התמונה. **שם הקובץ חייב להיות באנגלית בלבד** (למשל `rothschild-50.jpg`).
4. לוחץ "Commit changes".

### צעד 2: מוסיפים את הפרויקט ל-`projects.js`

1. פותח את `projects.js` במצב עריכה (אייקון עיפרון).
2. גולל לסוף המערך, ליד הסוגר `];`.
3. לפני הסוגר, מוסיף בלוק חדש:
   ```javascript
   {
     id: 'rothschild-50',
     name: 'רוטשילד 50',
     location: 'תל אביב',
     developer: 'שם היזם',
     units: '40 יח"ד',
     floors: '12 קומות',
     parking: '3 מרתפי חניה',
     area: '8,000 מ"ר',
     cover: 'assets/projects/rothschild-50.jpg',
     images: ['assets/projects/rothschild-50.jpg'],
   },
   ```
4. עורך את הערכים כך שיתאימו לפרויקט החדש שלך.
   - `id` — שם באנגלית, בלי רווחים, ייחודי.
   - `cover` ו-`images` — חייבים להצביע על קובץ התמונה שהעלית.
5. Commit changes.

### צעד 3 (אופציונלי): להציג גם בדף הבית

אם אתה רוצה שהפרויקט החדש יופיע בכרטיסיות בדף הבית, מצא בסוף `projects.js` את:
```javascript
window.FEATURED_IDS = [
  'yehuda-hanasi',
  'einstein-19',
  ...
];
```
והוסף בתוך המערך את ה-`id` החדש (`'rothschild-50',`).

## איך מחליפים תמונה של פרויקט קיים

1. תיקייה `assets/projects/` → "Add file" → "Upload files" → גורר את התמונה החדשה עם שם אנגלי **שונה** מהקיים (למשל `weizmann-105-v2.jpg`).
2. ב-`projects.js`, מוצא את הפרויקט ומעדכן את `cover` ואת `images` לנתיב החדש.

## איך מחזירים מצב אחורה אם משהו השתבש

GitHub שומר היסטוריה מלאה. כל שינוי הוא הפיך:

1. בדף הראשי של ה-repository, לחיצה על "commits" (מספר עם אייקון שעון).
2. מאתר את הקומיט האחרון שעבד טוב.
3. שלוש נקודות `...` → "Revert".
4. מאשר.

תוך דקה האתר חוזר למצב הקודם.
